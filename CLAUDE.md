# TEAMCCR Lab 웹사이트 — 프로젝트 지침

이 파일은 프로젝트의 모든 새 대화에 자동으로 주입됩니다. 아래 규칙을 항상 따르세요.

## 파일 구조
- **편집 마스터**: `TEAMCCR Prototype.dc.html` — 모든 수정은 이 파일(.dc.html)에서 한다.
- **배포/메인 파일**: `index.html` — 마스터의 배포용 사본. 마스터를 수정하면 항상 `index.html`도 동일하게 동기화한다.
- **런타임**: `support.js`, `image-slot.js` — 절대 직접 수정/삭제하지 않는다.
- **이미지**: `assets/` 폴더.

## 이미지 규칙 (중요)
- 모든 이미지는 `assets/` 폴더의 **실제 파일 + 링크** 방식으로 관리한다.
- 각 `image-slot`은 자기 `id`에 해당하는 파일을 `src`로 불러온다. 매핑은 마스터 JS의 `SLOT_IMAGES` 객체에 있다.
- 새 이미지를 추가할 때: `assets/`에 파일을 넣고 `SLOT_IMAGES`에 `'슬롯id':'./assets/파일명'` 항목을 추가한다.
- 파일명은 **소문자**로 통일한다 (Netlify는 리눅스 서버라 대소문자 구분).
- **base64 사이드카(`.image-slots.state.json`) 방식은 다시 쓰지 않는다.** 그 파일은 제거됐고, 실제 파일이 유일한 이미지 소스다.
- 이미지가 없는 슬롯은 `src` 없이 두면 자동으로 플레이스홀더가 표시된다.

## "익스포트" 요청 시
사용자가 "익스포트" 또는 "다운로드"라고 하면:
1. 그동안의 수정을 `index.html`에 반영(마스터 → index 동기화).
2. 배포에 필요한 파일 전체를 zip으로 묶어 다운로드로 제공한다.
- 배포 대상 파일: `index.html` + `support.js` + `image-slot.js` + `assets/` 폴더.
- 목적지: GitHub 커밋 → Netlify 정적 배포. (정적 웹서버라 이미지 링크가 정상 동작함.)

## 프로젝트 성격
- 리브리더(재호흡기) 중고 장비 마켓플레이스 모바일 프로토타입. 한국어 기본 + 다국어(EN/JA/ZH).
- 다크/라이트/시스템 화면 모드 지원. 폰 컬럼(max-width 430px) 레이아웃.
- iOS Safari 대응이 중요 (back 제스처, 팝업 위치, 스크롤 잠금 등).

## 팝업/오버레이 규칙 (iOS 이슈 재발 방지)
- 화면 전체를 덮는 팝업(설정 픽커, 지역/언어/화면모드, 외국인가입, 강사/지도/장바구니)은 **반드시 스크롤 컨테이너(`.phone-content`) 밖**, `#phone-col`의 직계 자식으로 둔다.
- 위치 스타일은 `_fixedOverlayStyle`(측정된 rect 기반, 뷰포트 고정)을 쓴다. `.phone-content` 안에 두면 다크모드 CSS 필터 때문에 스크롤 시 화면 밖으로 밀린다.
- 팝업 열 때 `_lock(true)`, 닫을 때 `_lock(false)`로 배경 스크롤을 잠근다.
